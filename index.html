Exercise 1: SELECT Basics

Easy

Write a query to select the name, email, and gpa of all students.

Expected Output
name | email | gpa -----------|-------------------|------ Ahmed | ahmed@nmu.edu.eg | 3.75 Sara | sara@nmu.edu.eg | 3.92 Mohamed | mohamed@nmu.edu.eg| 3.45

Exercise 2: WHERE Clause

Easy

Find all students with GPA greater than or equal to 3.5 who are majoring in 'Computer Science'.

Hint
-- Use AND to combine conditions SELECT * FROM students WHERE gpa >= 3.5 AND major = 'Computer Science';

Exercise 3: JOIN Tables

Medium

List all students along with the courses they are enrolled in. Show student name and course name.

Hint
-- Join students, enrollments, and courses SELECT s.name, c.course_name FROM students s JOIN enrollments e ON s.id = e.student_id JOIN courses c ON e.course_id = c.id;

Exercise 4: Aggregate Functions

Medium

Calculate the average GPA for each major and show only those with average GPA above 3.0.

Hint
-- Use GROUP BY and HAVING SELECT major, AVG(gpa) AS avg_gpa FROM students GROUP BY major HAVING AVG(gpa) > 3.0;

Exercise 5: Subquery

Hard

Find students who are NOT enrolled in any courses (hint: use NOT EXISTS or LEFT JOIN with IS NULL).

Solution
-- Using LEFT JOIN SELECT s.name FROM students s LEFT JOIN enrollments e ON s.id = e.student_id WHERE e.student_id IS NULL;

Challenge 1: Complex JOIN

Medium

Create a report showing: Student Name, Course Name, Professor Name, and Grade for all enrollments.

Tables needed: students, enrollments, courses, professors

Show Solution
SELECT s.name AS student, c.course_name, p.name AS professor, e.grade FROM students s JOIN enrollments e ON s.id = e.student_id JOIN courses c ON e.course_id = c.id JOIN professors p ON c.professor_id = p.id;

Challenge 2: Top Performers

Hard

Find the top 3 students with the highest average grade across all their courses.

Show Solution
SELECT s.name, AVG(CASE WHEN e.grade = 'A' THEN 4.0 WHEN e.grade = 'B' THEN 3.0 WHEN e.grade = 'C' THEN 2.0 ELSE 0 END) AS avg_grade_points FROM students s JOIN enrollments e ON s.id = e.student_id GROUP BY s.id, s.name ORDER BY avg_grade_points DESC LIMIT 3;

Challenge 3: Course Statistics

Hard

For each course, show: course name, total enrollments, number of A grades, and percentage of students who got A.

Show Solution
SELECT c.course_name, COUNT(e.student_id) AS total_enrollments, SUM(CASE WHEN e.grade = 'A' THEN 1 ELSE 0 END) AS a_grades, ROUND( SUM(CASE WHEN e.grade = 'A' THEN 1 ELSE 0 END) * 100.0 / COUNT(e.student_id), 2 ) AS a_percentage FROM courses c LEFT JOIN enrollments e ON c.id = e.course_id GROUP BY c.id, c.course_name;

Challenge 4: Recursive CTE (Advanced)

Expert

Assuming you have an employees table with a manager_id column, write a query to show the complete hierarchy from CEO down to all employees.

This requires Common Table Expressions (CTEs) - an advanced MySQL feature.

Show Solution
WITH RECURSIVE employee_hierarchy AS ( -- Base case: CEO (no manager) SELECT id, name, manager_id, 0 AS level FROM employees WHERE manager_id IS NULL UNION ALL -- Recursive case: employees with managers SELECT e.id, e.name, e.manager_id, eh.level + 1 FROM employees e JOIN employee_hierarchy eh ON e.manager_id = eh.id ) SELECT REPEAT(' ', level) || name AS hierarchy FROM employee_hierarchy ORDER BY level, name;

Step 3: Schema Design

Convert your ER diagram into actual database tables with columns and data types.

بالعربي: تصميم المخطط (Schema)
دلوقتي هنحول الـ ER Diagram لجداول حقيقية. حدد نوع البيانات لكل عمود (INT, VARCHAR, DATE) والمفاتيح (Primary Keys, Foreign Keys).

Example Schema:

Library Database Schema
CREATE TABLE students ( student_id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE NOT NULL, phone VARCHAR(20), major VARCHAR(50), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE TABLE books ( book_id INT AUTO_INCREMENT PRIMARY KEY, title VARCHAR(200) NOT NULL, author VARCHAR(100), isbn VARCHAR(20) UNIQUE, category VARCHAR(50), available BOOLEAN DEFAULT TRUE ); CREATE TABLE borrowings ( borrowing_id INT AUTO_INCREMENT PRIMARY KEY, student_id INT, book_id INT, borrow_date DATE, return_date DATE, FOREIGN KEY (student_id) REFERENCES students(student_id), FOREIGN KEY (book_id) REFERENCES books(book_id) );

Data Types Guide:

Data Type Use For Example
INT Whole numbers ID, Age, Count
VARCHAR(n) Text up to n chars Name, Email
DECIMAL(p,s) Exact decimal Price (10,2)
DATE/DATETIME Dates and times Birth date
BOOLEAN True/False Is active

Step 4: Normalization

Organize your database to reduce redundancy and improve data integrity.

بالعربي: التسوية (Normalization)
التسوية بتساعدنا نمنع تكرار البيانات. في 3 مستويات (1NF, 2NF, 3NF). كل مستوى بيحل مشكلة معينة في تصميم الجدول.

Normal Forms:

1NF - First Normal Form

Each cell contains a single value (no repeating groups)

كل خلية ليها قيمة واحدة بس، مفيش قوائم جوة خلية

2NF - Second Normal Form

No partial dependencies (non-key columns depend on entire primary key)

كل عمود مش مفتاح لازم يعتمد على المفتاح الأساسي كله

3NF - Third Normal Form

No transitive dependencies (columns only depend on primary key)

مفيش عمود يعتمد على عمود تاني مش مفتاح

Before & After Normalization
-- NOT Normalized (Bad Design) CREATE TABLE orders_bad ( order_id INT, customer_name VARCHAR(100), -- Should be in separate table customer_email VARCHAR(100), -- Redundant customer info product_name VARCHAR(100), -- Should be in separate table product_price DECIMAL(10,2), -- Product info repeated quantity INT ); -- Normalized (Good Design) CREATE TABLE customers ( customer_id INT PRIMARY KEY, name VARCHAR(100), email VARCHAR(100) ); CREATE TABLE products ( product_id INT PRIMARY KEY, name VARCHAR(100), price DECIMAL(10,2) ); CREATE TABLE orders ( order_id INT PRIMARY KEY, customer_id INT, order_date DATE, FOREIGN KEY (customer_id) REFERENCES customers(customer_id) ); CREATE TABLE order_items ( order_item_id INT PRIMARY KEY, order_id INT, product_id INT, quantity INT, FOREIGN KEY (order_id) REFERENCES orders(order_id), FOREIGN KEY (product_id) REFERENCES products(product_id) );

Step 5: Create Tables

Write the actual SQL CREATE TABLE statements to build your database.

بالعربي: إنشاء الجداول
دلوقتي هنكتب أوامر SQL عشان نعمل الجداول. متنساش تعمل قاعدة البيانات الأول (CREATE DATABASE) وبعدين تستخدمها (USE).

Step-by-Step SQL:

Complete Example: Library Database
-- Step 1: Create Database CREATE DATABASE IF NOT EXISTS library_db; USE library_db; -- Step 2: Create Students Table CREATE TABLE students ( student_id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE NOT NULL, phone VARCHAR(20), major VARCHAR(50), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Step 3: Create Books Table CREATE TABLE books ( book_id INT AUTO_INCREMENT PRIMARY KEY, title VARCHAR(200) NOT NULL, author VARCHAR(100), isbn VARCHAR(20) UNIQUE, category VARCHAR(50), available BOOLEAN DEFAULT TRUE ); -- Step 4: Create Borrowings Table (with foreign keys) CREATE TABLE borrowings ( borrowing_id INT AUTO_INCREMENT PRIMARY KEY, student_id INT NOT NULL, book_id INT NOT NULL, borrow_date DATE NOT NULL, return_date DATE, status VARCHAR(20) DEFAULT 'borrowed', FOREIGN KEY (student_id) REFERENCES students(student_id) ON DELETE CASCADE ON UPDATE CASCADE, FOREIGN KEY (book_id) REFERENCES books(book_id) ON DELETE CASCADE ON UPDATE CASCADE );
Pro Tips:
  • Always use IF NOT EXISTS to avoid errors
  • Define PRIMARY KEY for every table
  • Use appropriate data types for each column
  • Set foreign keys to maintain referential integrity
  • Use ON DELETE CASCADE carefully (deletes related data)

Step 6: Set Up Relationships

Implement foreign keys and establish connections between your tables.

بالعربي: إعداد العلاقات
العلاقات بين الجداول بتعملها من خلال Foreign Keys. الـ Foreign Key في الجدول الفرعي بيشاور على الـ Primary Key في الجدول الرئيسي.

Types of Relationships:

One-to-One

One record in Table A relates to one record in Table B

Example: User ↔ UserProfile

One-to-Many

One record in Table A relates to many in Table B

Example: Department ↔ Employees

Many-to-Many

Many records in A relate to many in B (needs junction table)

Example: Students ↔ Courses

Foreign Key Examples
-- One-to-Many: One student has many borrowings CREATE TABLE borrowings ( borrowing_id INT PRIMARY KEY, student_id INT, book_id INT, FOREIGN KEY (student_id) REFERENCES students(student_id), FOREIGN KEY (book_id) REFERENCES books(book_id) ); -- Many-to-Many: Students enroll in many Courses -- Need a junction table: enrollments CREATE TABLE enrollments ( enrollment_id INT PRIMARY KEY, student_id INT, course_id INT, semester VARCHAR(20), FOREIGN KEY (student_id) REFERENCES students(student_id), FOREIGN KEY (course_id) REFERENCES courses(course_id) );

Step 7: Test & Optimize

Verify your database works correctly and optimize for performance.

بالعربي: اختبار وتحسين
جرب قاعدة البيانات بتاعتك: ضيف بيانات، جرب الاستعلامات، تأكد إن العلاقات شغالة صح. بعدين اعمل Indexes عشان تسريع البحث.

Testing Checklist:

Optimization: Creating Indexes
-- Index for frequently searched columns CREATE INDEX idx_student_email ON students(email); CREATE INDEX idx_book_title ON books(title); CREATE INDEX idx_borrowing_dates ON borrowings(borrow_date, return_date); -- Composite index for common query patterns CREATE INDEX idx_book_category_available ON books(category, available); -- Check if queries use indexes EXPLAIN SELECT * FROM students WHERE email = 'test@example.com';
